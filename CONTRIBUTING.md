# Contributing

感谢你对 `Lark-Formatter` 的关注。为避免开源发布时出现版权、隐私或许可证问题，请在提交前遵守以下规则。

## 1. 贡献默认许可
- 除非你在提交中另行明确说明，向本项目提交的源码、文档与测试代码默认按项目当前许可证 `MIT` 贡献。
- 你提交的内容必须是你本人原创，或你已确认拥有合法的再分发权。

## 2. 可以提交的内容
- 项目源码、脚本、测试代码、文档。
- 你本人原创的图标、说明文档、示例配置。
- 代码动态生成的最小化测试夹具。
- 你本人原创且已脱敏、可公开再分发的 synthetic / public fixtures。

## 3. 不接受提交的内容
以下内容请仅保留在本地私有环境，不要提交到公开仓库：
- 高校 / 期刊 / 机构的格式规范原文、摘录、提取文本或改写副本。
- 真实论文、真实报告、真实客户文档，或包含真实姓名 / 学号 / 学校 / 单位 / 课题信息的样本。
- 运行生成的 `*_new.docx`、`*_对比稿.docx`、审计报告、回归日志、临时目录。
- 来源不明或未确认再分发许可的品牌图标、模板、封面素材。
- 从第三方文档中抽取出的 prompt 中间产物，例如 `prompt_filled.txt`、`spec_extracted.txt`、`summary.json` 等。

## 4. 测试素材要求
- 公开测试素材统一优先放在 `tests/fixtures_public/`。
- 测试素材必须满足：
  - 可公开再分发；
  - 不含第三方规范正文；
  - 不含真实个人 / 机构信息；
  - 尽量最小化，仅保留复现问题所需内容。
- 如无法明确证明素材可公开，请默认不要上传。

## 5. 提交前自检
提交 PR 前建议至少完成以下检查：
- 运行测试：`python -m pytest -q`
- 运行公开发布扫描：`python scripts/check_public_release.py --strict`
- 检查新增文件是否需要同步更新：
  - `THIRD_PARTY_NOTICES.md`
  - `docs/OPEN_SOURCE_MIT_RELEASE_CHECKLIST.md`
  - `docs/NOT_FOR_UPLOAD.md`

## 6. 关于第三方依赖
- 本项目源码采用 `MIT`，但第三方依赖不会因此变成 `MIT`。
- 当前桌面端依赖 `PySide6`，如你分发打包后的二进制，请自行确认 Qt / PySide6 的许可证履约要求。

如你不确定某份文档、图标、模板或样本是否可以公开，请先不要提交。
